# Skill: release_prep

## 1. 适用场景

- 用户明确授权进入发布准备阶段。
- 需要整理发布清单、上线前验证项、回滚与风险说明。

## 2. 非适用场景

- 常规开发、调试、文档更新阶段。
- 未明确收到“允许发布/进入上线准备”指令。

## 3. 必读 Authority Docs

1. `AGENT_ENTRYPOINT.md`
2. `docs/V0_9_MODEL_HANDOFF_MANIFEST.md`
3. `docs/V0_9_BACKEND_SOURCE_OF_TRUTH.md`
4. `docs/api/openapi.yaml`
5. 与变更相关的 skill 文档

## 4. 禁止事项

- 禁止在未授权情况下执行任何发布相关动作。
- 禁止把“本地验证通过”表述为“线上已生效”。
- 禁止跳过人工 review 直接进入发布。

## 5. 最小实施原则

- 仅输出当前轮次所需的发布准备信息，不做平台化流程改造。
- 把发布前条件、发布步骤、回滚步骤写成可执行清单。
- 明确区分：本地验证、预发验证、线上验证。

## 6. 必跑测试命令

至少复核：

- `go build ./cmd/server`
- 与本轮改动直接相关的 `go test` 集合

如环境允许，再补充端到端或 smoke 检查命令。

## 7. 文档更新要求

- 发布准备文档必须注明授权来源与时间。
- 若发布流程有新增约束，更新 `docs/ENGINEERING_RULES.md` 的发布部分。
- 保持 authority 文档与运行时一致。

## 8. 标准输出格式要求

按 `templates/agent_output.md`，并在“发布结果”中明确：

- 是否已授权
- 是否已执行发布动作
- 发布后验证状态

## 9. 是否允许进入 release prep

- 本 skill 即发布准备 skill。
- 但仍需外部明确授权，未授权视为不允许。

## 10. 常见误区 / 历史包袱提醒

- “默认发布”是本工程已废弃模式。
- 发布准备不等于实际发布，输出中必须区分。
